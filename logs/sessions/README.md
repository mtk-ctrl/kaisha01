# 作業ログ — Jobs (AI CEO) 活動記録

オーナーが「Jobsが何をしたか」を日本語で確認するための専用フォルダです。

---

## 目的

- **透明性**: Jobsが何をいつやったかをオーナーが一目で追える
- **引き継ぎ**: 新しいセッションを開始したJobsが前回の続きを把握できる
- **説明責任**: 「やります」と言ったことの実行証跡を残す

---

## 命名規則

```
YYYY-MM-DD_NNN_slug.md
```

| 要素 | 説明 |
|------|------|
| `YYYY-MM-DD` | 作業日 |
| `NNN` | その日の通し番号（001〜） |
| `slug` | 作業内容を表す英小文字スラッグ |

例: `2026-05-04_001_corporate-website.md`

---

## 記録ルール（Jobs 必須事項）

1. **セッション中に大きな作業が完了したら必ずログを作成する**
2. ログには以下を必ず記載する:
   - 日時・セッションID（わかる場合）
   - 何をしたか（日本語箇条書き）
   - なぜしたか（背景・判断理由）
   - 変更したファイル/URL
   - 次のアクション
3. CLAUDE.md のセッション開始ルーティンにも「ログ確認」を追加済み
4. コード変更を伴う場合は、対応するGitコミットハッシュも記載する
