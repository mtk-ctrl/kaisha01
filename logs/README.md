# logs/ — 記録の置き場

オーナーが「Jobsが何をしたか」を日本語で追うためのフォルダです。

> **2026-06-16 運用変更**: セッションごとの作業ログ（`logs/sessions/`・`YYYY-MM-DD_NNN_slug.md`）は**廃止・削除しました**。
> ログが二重三重になり、毎セッション読む負荷が高かったため、下記のシンプルな3層に集約しています。

---

## 記録の3層（どこに何を書くか）

| 層 | 場所 | 内容 |
|----|------|------|
| **一次ログ** | git コミットメッセージ | 何をしたか・なぜしたかを毎タスク記載（これが正） |
| **直近の流れ** | `NOW.md` ⑤ | 直近8件だけ1行サマリ。新セッションが現在地を掴むため |
| **全履歴（記録庫）** | `logs/CHANGELOG.md` | 過去の完了タスク全部。**必要なときだけ読む** |
| **重要な経営判断** | `logs/decisions/YYYY-MM-DD_topic.md` | 方針転換・構想など、後から参照したい判断のみ |

---

## ルール

1. タスク完了のたびに **コミットメッセージ ＋ NOW.md ⑤ に1行**。
2. NOW.md ⑤ は直近8件のみ。あふれた分は `logs/CHANGELOG.md` の先頭へ移す。
3. 重要な経営判断は `logs/decisions/` に独立ファイルで。
4. `logs/sessions/` は作らない（廃止済み）。

---

*このフォルダはJobsが自律的に更新します。オーナーは読むだけでOK。*
