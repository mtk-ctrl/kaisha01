## 判断の記録

**パスワードをクライアントサイドにした理由:**
バックエンドなしで即実装できる。現段階は完全なセキュリティより「軽いアクセス制限」が目的のため適切。将来的に API Route + セッション管理に移行可能。

**外部アプリを iframe でなく新タブにした理由:**
GitHub Pages のサイトは X-Frame-Options による iframe ブロックの可能性がある。新タブ（`target="_blank"`）なら確実に動作する。
