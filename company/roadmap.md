## 🎯 North Star（最終的な理想状態）

**「小4の春に TANQ だけで学び始めた子が、小6の秋に偏差値60に届いている。子どもは毎日15分の日課を自分から続けており、親はその成長を毎週レポートで見て、納得して月980円を払い続けている。」**

- 上位互換の既決事項: 「このサービスだけで中受偏差値60」（`logs/decisions/2026-06-11_chuju-vision.md`）
- 判断基準は不変: ①小4が「おもしろい」と思うか ②親が課金したくなるか

## ✅ 理想状態の7条件（検証可能な Done の定義）

| # | 条件 | 満たすべき状態 | 現在地（2026-07） |
|---|------|--------------|------------------|
| C1 | 網羅 | 4教科 約80単元すべてに「まなぶ→とく→たしかめ」 | 骨格完成・「たしかめ」0% |
| C2 | 深さ | 各単元 基本→練習→応用の3層 20〜25問 | 12〜15問/単元 |
| C3 | 定着 | 解いた単元が忘れる頃に再出題（SRS横断復習） | 漢字・英語のみ。中受単元ゼロ |
| C4 | 測定 | 月例の範囲混合テスト＋内部偏差値換算で到達度が見える | ゼロ |
| C5 | 子の継続 | 診断ミッション＋相棒＋コイン経済で日課15分×週5が自走 | 土台のみ（ミッション機械的選定・コイン使い道4種） |
| C6 | 親の価値 | 週次AIレポート＋声かけ提案が届く | ゼロ |
| C7 | 証明 | 実測データで「偏差値60」を外部に示せる | スコア収集基盤のみ（実証データなし） |

**「問題が次々出る」根本原因の総括**: これまでの完成判定は「単元が公開されたか」であり、C1〜C7への寄与に紐づいていなかった。以後、各案件は「どの条件を何%進めるか」を明示して着手する（テツの草案・Jobs＋アキのレビューで必須項目）。

## 🗺️ ロードマップ（4ステージ・出口ゲート制）

期日は目安。**ゲート＝出口条件の充足**で通過を判定し、細切れ原則（質＞スピード）を優先する。

### Stage 1「深さ」— 解くに値する中身（〜2026年9月目安）＝ C2・C1前半

承認済みの優先順位（`logs/decisions/2026-07-11_content-expansion-priority.md`）そのもの:
1. 国語読解の本格化（ステップ4=長文800〜1200字）
2. 既存単元の演習量3層化（20〜25問/単元）
3. 算数 立体図形・ダイヤグラム（diagramType 判断先行）
4. 理科知識21単元への「まなぶ」追加
5. 社会 地理続き（工業・貿易・地形図）

**出口ゲート**: 主要単元が3層20問超／読解に長文が揃う／4教科の「まなぶ→とく」に品質差がない／「正解=最長」バイアス解消（触った単元から）

### Stage 2「定着と測定」— 力がつく・見える（〜2026年11月目安）＝ C1完・C3・C4

0. **単元×正答率の集計基盤**（scores/tester_scores は収集済み。Stage 3 の診断も同じ基盤に載る）
1. 各単元に「たしかめ」（合格判定つき確認テスト）＝ まなぶ→とく→たしかめ の3段完成
2. 中受単元へのSRS横断復習（忘れる頃に再出題）
3. 月例・範囲混合の到達度テスト＋内部偏差値換算基準 v1

**出口ゲート**: 公開全単元に「たしかめ」あり／子どもの画面に「今の到達度」が出る

### Stage 3「個別化と親価値」— 続く・売れる（〜2027年1月目安）＝ C5・C6

1. AIつまずき診断ミッション（`missions.ts` の `generateMissions()` 置換。置換境界は設計済み）
2. ごほうび経済の拡充（コインの使い道: きせかえ・部屋・コレクション。射幸性なしの買い切りのみ）
3. 親向け週次AIレポート＋声かけ提案（課金継続の生命線）
4. 料金プランの実装・課金開始（アキ判断: C6が動いてから課金する方が解約率で有利）

**出口ゲート**: 日課ループの継続データが取れている／課金導線が動く

### Stage 4「証明と拡大」— 信じられる・広がる（2027年〜）＝ C7＋事業目標

1. テスター拡大 → 実測データで「TANQ利用量×成績」の相関検証（C7）
2. マーケ始動（ハナ: X・note）
3. タロウ召喚（B2B: 塾・学童プラン）

**計器盤**: `company/monetization.md` の既定路線（有料200名 → MRR50万円）

## 🔁 運用フライホイール（全ステージ共通）

```
コンテンツを出す → スコアデータが貯まる → 弱点（子の・コンテンツの）が見える
   ↑                                                    ↓
オーナー判定 ← 画面レビュー ← 細切れで改善（質＞スピード）
```

- 品質原則は不変: `product/hint-design.md` 準拠・Ken全問検証・コード＋画面レビュー・正直表示（近日公開を偽らない）

## 🚫 やらないこと（既決の再確認）

- 単発アプリの横展開（30本目を作らない）
- 市販文章の転載（AI生成オリジナル文で代替）
- 最難関（筑駒・灘等）対応を謳うこと
- 射幸性のあるごほうび（ガチャ等）

## 📏 この文書の使い方

- 新しい案件・提案はまず「7条件のどれを進めるか」を明示する（該当なしなら原則やらない）
- ステージ通過時に Jobs が本文書の「現在地」列とゲート判定を更新し、`logs/decisions/` に通過記録を残す
- North Star・7条件そのものの変更はオーナー承認事項
