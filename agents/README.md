## 案件フロー（2026-05-18 改訂）

```
オーナー（取締役会）
    ↓ 案件・指示
Jobs
  ├─ 誰が何をやるかを判断して役割を割り振る（Jobsの仕事）
  ├─ テツ → アイデア草案・方針検討
  ├─ リン / Ken / ソラ / Sarah / ゆい → 各専門観点で参加
  └─ テツを含む全員で対等に議論
         ↓ 議論まとめ（テツがJobsとアキへ上申）
Jobs + アキ（レビュー・ブラッシュアップ・差し戻し）
         ↓ ← 必要なら差し戻して再議論を繰り返す
       GO
         ↓
実装（テツが主導）
         ↓
Jobs（完了確認・オーナーへ報告）
```

## エージェント召喚ルール

各エージェントは Claude Code の Agent ツールで召喚する。  
召喚時に対応する `agents/*.md` の内容をプロンプトに含めること。

```
案件受け取り時の標準手順:
1. Jobs が案件の性質を見て「誰が参加するか」を決める
2. テツを召喚 → agents/tetsu.md を読み込み → アイデア草案を依頼
3. 必要なスタッフ（リン/Ken/ソラ等）を召喚 → テツと議論させる
4. テツが議論まとめを出す
5. アキを召喚 → agents/aki.md を読み込み → Jobsと共にレビュー
6. Jobs が最終GO/NO-GO判断
```

## インフラ（スマホ運用含む）

**オーナーがスマホから Claude に指示するだけで本番デプロイまで完結する。**

```
オーナー（スマホ）
    ↓ Claude に指示
Jobs（Claude Code がコードを変更・commit・push）
    ↓ claude/* ブランチへ push
GitHub Actions（自動起動）
    ├─ main に自動マージ
    └─ Vercel に本番デプロイ（VERCEL_TOKEN で認証）
         ↓
tanq-app.vercel.app に反映
```

**オーナーが直接操作する必要があるもの: なし**  
（PowerShell・CLI・Vercel管理画面への操作は一切不要）
