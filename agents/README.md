## 案件フロー（新体制）

```
オーナー（取締役会）
    ↓ 案件・指示
Jobs（受け取り → テツへ割り振り）
    ↓
テツ（初期分析・選択肢整理・スタッフへ振り分け）
    ↓
    ├── アキ  → 事業・設計レビュー（GO/NO-GO）
    ├── Sarah → 保護者UXレビュー（OK/要修正/NG）
    ├── リン  → 子どもUXテスト
    ├── ゆい  → 低学年UXテスト
    ├── Ken  → コンテンツ品質確認
    ├── ソラ  → データ分析・KPI
    └── ハナ  → マーケ判断
    ↓
テツ（議論まとめ → Jobs へ上申）
    ↓
Jobs（最終評価・GO/NO-GO → テツへ実装指示）
    ↓
テツ（実装主導）
    ↓
Jobs（完了確認・オーナーへ報告）
```

## エージェント召喚ルール

各エージェントは Claude Code の Agent ツールで召喚する。  
召喚時に対応する `agents/*.md` の内容をプロンプトに含めること。

```
案件受け取り時の例:
1. テツに振る → agents/tetsu.md を読み込み → 初期分析を依頼
2. テツの振り先に応じて → agents/aki.md / sarah.md / rin.md 等を読み込み → 議論させる
3. テツの議論まとめを受けて Jobs が最終判断
```
